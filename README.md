# sp-homework

## Описание:

Проект sp-homework - это блоки кода из домашнего задания. Содержит генераторы, утилиты чтения данных, обработку, поиск и интерактивный интерфейс для работы с банковскими транзакциями.

### Описание файла generators.py

Данный файл содержит три полезные функции для работы с финансовыми транзакциями и генераторами:

Функции:

1. `filter_by_currency(transactions, currency_code)`
- Назначение: фильтрует транзакции по указанному коду валюты.
- Входные аргументы:
    * transactions: список транзакций, представленных в виде словарей.
    * currency_code: код интересующей валюты (например, "USD", "EUR", "RUB").
- Выход: итератор, выдающий только те транзакции, которые соответствуют указанной валюте.
2. `transaction_descriptions(transactions)`
- Назначение: извлекает описания всех транзакций из предоставленного списка.
- Входные аргументы:
    * transactions: список транзакций, представленных в виде словарей.
- Выход: итератор, выдающий описание каждой транзакции.
3. `card_number_generator(start_position, end_position)`
- Назначение: генерирует номера банковских карт в заданном диапазоне.
- Входные аргументы:
    * start_position: начальное значение диапазона.
    * end_position: конечное значение диапазона.
- Выход: итератор, выдающий номера карт в формате XXXX XXXX XXXX XXXX.


Пример использования

```python
# Фильтруем транзакции по валюте USD
filtered_transactions = list(filter_by_currency(transactions, "USD"))

# Получаем описания транзакций
descriptions = list(transaction_descriptions(filtered_transactions))

# Генерация карточных номеров от 1 до 5
generated_cards = list(card_number_generator(1, 5))

# Вывод результатов
for desc in descriptions:
    print(desc)

for card in generated_cards:
    print(card)
```


### Описание файла decorators.py

Данный модуль реализует декоратор `log`, предназначенный для ведения подробного логирования выполняемых функций, отслеживая начало и завершение их выполнения, а также регистрируя результат или возникающие ошибки.

#### Установка зависимостей
Модуль не требует внешних зависимостей и основан исключительно на стандартной библиотеке Python.

#### Особенности
- Автоматическое создание логов при старте и завершении выполнения функции.
- Регистрация успешных результатов выполнения функций.
- Отображение возникающих ошибок и трассировки стека.
- Возможность выбора места хранения логов: в файл или в консоль.

#### Параметры декоратора
| Параметр     | Значение                                   |
|--------------|-------------------------------------------|
| `filename`   | Необязательный аргумент. Путь к файлу, куда будут сохранены логи. Если не задан, логи выводятся в консоль. |

#### Использование
Подключите модуль и используйте декоратор следующим образом:

```python
from src.decorators import log

@log(filename="mylog.txt")
def calculate_sum(a, b):
    return a + b

calculate_sum(2, 3)
```

Если аргумент filename не передан, логи будут отправлены в консоль
```python
from src.decorators import log

@log()
def greet(name):
    return f"Hello, {name}!"

greet("Alice")
```

### Описание файла operations_loader.py
Данный файл представляет собой утилиту для чтения данных из файлов формата CSV и Excel (XLS, XLSX). Основная цель — обеспечить удобное извлечение данных из указанных форматов и представление их в виде списков словарей для дальнейшего анализа и обработки.

Проект состоит из трех ключевых компонентов:
- read_csv: функция для чтения CSV-файлов.
- read_xlsx: функция для чтения Excel-файлов.
- read_operations: главная функция, определяющая формат файла и выбирающая нужную функцию для чтения.

#### Установка зависимостей
Для запуска приложения необходимы следующие пакеты Python:
- pandas >= 2.0
- openpyxl (для работы с .xlsx)
# для поддержки .xls при необходимости:
# - xlrd==1.2.0
Установка зависимостей осуществляется следующей командой:
```
pip install pandas openpyxl
```

### Описание файла processing.py
Модуль обработки и поиска по операциям.

Функции:
- filter_by_state(data_list, state="EXECUTED") — фильтрация по статусу.
- sort_by_date(data_list, reverse=True) — сортировка по дате ISO-формата.
- process_bank_search(data: list[dict], search: str) -> list[dict] — поиск операций по подстроке в поле description (регистронезависимо, с использованием `re`).
- process_bank_operations(data: list[dict], categories: list[str]) -> dict[str, int] — подсчёт количества операций по списку категорий (сопоставление по подстроке description, регистронезависимо, использует collections.Counter).

### Описание файла main.py
Интерактивная точка входа. Предоставляет пользователю меню выбора источника данных (JSON/CSV/XLSX), фильтрацию по статусу (с нормализацией регистра), опциональную сортировку по дате (по возрастанию/убыванию), фильтрацию только рублёвых операций и поиск по слову в описании. Итоговые операции выводятся в консоль в читаемом виде.
